if setting the cookie in the browser does not work then do the following

backend:
cors({
    origin:"http://localhost:5173",
    credentials:true
})

const token = await jwt.sign({id:user._id,email:user.email},process.env.JWT_SECRET!,{expiresIn:"7d"}) //always include expiresIn field,indicates the availability of token for the time specified in the expiresIn property.

console.log(token)

res.cookie("token",token,{sameSite:"lax",secure:false,maxAge: 7 * 24 * 60 * 60 * 1000}).json({message:'successfully signed in',userinfo:{username:user.username,email:user.email}})

always include this sameSite,secure,maxAge property and set them exactly. You can set sameSite:none and secure:true in production.If maxAge is not set,then cookie is going to disappear from the browser when website is closed.
